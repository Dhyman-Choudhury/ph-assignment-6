Frequently Asked Questions

1. What is the difference between var, let, and const?
var: Function-scoped, hoisted with undefined, allows redeclaration and reassignment.
let: Block-scoped, hoisted but not initialized, allows reassignment but not redeclaration.
const: Block-scoped, hoisted but not initialized, does not allow reassignment or redeclaration.
2. What the difference between map(), forEach(), and filter()?
map(): Creates a new array by transforming elements.
forEach(): Loops through elements but does not return a new array.
filter(): Creates a new array with elements that pass a condition.
3. What are arrow functions and how they are different from regular functions?

Syntax – Arrow functions are shorter, while regular functions use function.

this Binding – Regular functions have their own this, but arrow functions inherit this from their surrounding scope.

Arguments Object – Regular functions have an arguments object, whereas arrow functions do not.

Constructor Support – Regular functions can be used with new, but arrow functions cannot.

4. How JavaScript Promises work?

A JavaScript Promise is an object that represents the eventual completion or failure of an asynchronous operation. It has three states: Pending (still in progress), Fulfilled (operation successful), and Rejected (operation failed). Promises are handled using `.then()` for success, `.catch()` for failure, and `.finally()` for cleanup, allowing you to write asynchronous code more easily.

5. How closures work in JavaScript?

A closure in JavaScript occurs when a function retains access to its lexical scope, even after the function that created it has finished executing. This means a function can remember and access variables from its outer (enclosing) function even after the outer function has completed. Closures are commonly used for data encapsulation, private variables, and creating functions with specific behavior.
